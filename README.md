
# GitBlox

## What is GitBlox?

GitBlox is a centralized version control and collaboration platform designed for Roblox developers. It enables you to make your projects open-source and collaborate with the community seamlessly.

---

## Features

- **Studio Plugin**: A user-friendly Git interface built directly into Roblox Studio - no terminal knowledge required
- **Web Dashboard**: Browse, discover, and contribute to open-source Roblox projects
- **Asset Inspector**: Visualize your game's structure and track changes across versions
- **Zero Setup**: Load your game with a unique project key and start collaborating instantly
- **100% Free**: No paywalls, no premium tiers, no data selling
- **Open Source**: Full transparency - inspect the code yourself

## Important: Is Open-Source Right for Your Game?

**GitBlox is designed for developers who want to go open-source.** Before you do, understand the trade-offs:

### Security & Theft Risks
- **Exploiters can see your code** - This means malicious actors can analyze your anti-cheat, exploit detection, and security systems to find vulnerabilities
- **Your game can be stolen** - Anyone can fork your repository and claim your work as their own. While you own the copyright, enforcement is difficult
- **No protection for unfinished games** - If you open-source early, others can take your incomplete work

### When Open-Source Makes Sense
✅ Games you've already released and stabilized  
✅ Games where security isn't critical (puzzle games, tycoons, etc.)  
✅ Educational projects and learning resources  
✅ Games where community contributions outweigh the risks  

### When to Keep Your Game Private
❌ Games still in development  
❌ Games with valuable proprietary mechanics  
❌ Games with sensitive anti-cheat or security systems  
❌ Games where your unique gameplay is the main value  

**GitBlox supports both public and private repositories** - you choose what works best for your project.

## FAQ

### Is GitBlox a virus?

No. GitBlox is **completely open-source** - both the Studio plugin and web platform are publicly available on GitHub, so anyone can inspect the code. There's nothing to hide.

### Can exploiters see my code?

Yes, if you make your repository public. Your scripts, anti-cheat logic, and security systems will be visible. This is a fundamental trade-off of open-source development. Use **private repositories** if you want to keep your code hidden.

### Can someone steal my game?

Technically, yes. Anyone can copy your game by downloading the `.rbxl` place file, or copying your code. However, **you own the copyright** - the original repository and commit history prove you created it first. If theft occurs, you can file DMCA claims or report to Roblox. That said, prevention is easier than enforcement.

### Is GitBlox free?

Yes. GitBlox is **100% free** with no hidden costs or premium tiers.

### What about my data?

We're based in the EU and comply with strict data protection laws. We don't sell user data, and your repositories remain under your control.

### Do I need to know Git?

Not at all. The Studio plugin handles 90% of Git commands with a simple, friendly interface. If you want advanced Git features, there's also a terminal for power users.

### Can I collaborate in real-time?

GitBlox uses asynchronous collaboration (pull requests and commits) rather than live co-editing, due to Roblox ToS limitations. However, multiple developers can work on the same project through version control.

### Can I keep my game private?

Yes. GitBlox supports **both public and private repositories**. Keep your game private while you develop, then open-source it when you're ready.
